PURPOSE

ML_OPTIONS M-file for ML_options.fig

SYNOPSIS

function varargout = ML_options(varargin)

DESCRIPTION 

 ML_OPTIONS M-file for ML_options.fig
      ML_OPTIONS, by itself, creates a new ML_OPTIONS or raises the existing
      singleton*.

      H = ML_OPTIONS returns the handle to a new ML_OPTIONS or the handle to
      the existing singleton*.

      ML_OPTIONS('CALLBACK',hObject,eventData,handles,...) calls the local
      function named CALLBACK in ML_OPTIONS.M with the given input arguments.

      ML_OPTIONS('Property','Value',...) creates a new ML_OPTIONS or raises the
      existing singleton*.  Starting from the left, property value pairs are
      applied to the GUI before ML_options_OpeningFcn gets called.  An
      unrecognized property name or invalid value makes property application
      stop.  All inputs are passed to ML_options_OpeningFcn via varargin.

      *See GUI Options on GUIDE's Tools menu.  Choose "GUI allows only one
      instance to run (singleton)".

 See also: GUIDE, GUIDATA, GUIHANDLES
